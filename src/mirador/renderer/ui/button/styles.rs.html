1//! # Button Styles Module
2//!
3//! This module provides predefined button styles for a user interface application.
4//! It implements a cohesive design system with semantic color variants and consistent
5//! styling parameters across different button types.
6//!
7//! ## Design Philosophy
8//!
9//! The button styles follow modern design principles:
10//! - **Semantic Colors**: Each button type (primary, warning, danger) uses colors
11//!   that communicate their intended purpose to users
12//! - **Accessibility**: High contrast ratios between text and background colors
13//! - **Consistency**: Uniform padding, border radius, and typography across all styles
14//! - **Interactive States**: Distinct visual feedback for hover, pressed, and disabled states
15//! - **DPI Awareness**: Automatic scaling based on display density
16//!
17//! ## Color Palette
18//!
19//! The module uses a professional slate-based color scheme with semantic variants:
20//! - **Primary (Green)**: For primary actions and positive confirmations
21//! - **Warning (Orange)**: For actions that require caution
22//! - **Danger (Red)**: For destructive or irreversible actions
23//!
24//! ## Usage Example
25//!
26//! ```rust
27//! use crate::button_styles::{create_primary_button_style, create_warning_button_style};
28//!
29//! let primary_style = create_primary_button_style();
30//! let warning_style = create_warning_button_style();
31//! ```

41/// Creates a primary button style with a mint green color scheme.
42///
43/// Primary buttons are used for the main action on a screen or form.
44/// They should be used sparingly - typically only one primary button per view.
45///
46/// ## Visual Characteristics
47/// - **Base Color**: Dark mint green (`rgb(30, 110, 30)`)
48/// - **Hover State**: Darker mint green for visual feedback
49/// - **Pressed State**: Darkest mint green to indicate active press
50/// - **Disabled State**: Muted light green to indicate unavailability
51/// - **Typography**: Medium weight, white text for high contrast
52///
53/// ## Design Rationale
54/// The green color palette communicates positive action and success.
55/// The slightly desaturated tones provide a professional appearance while
56/// maintaining accessibility standards.
57///
58/// ## Returns
59/// A `ButtonStyle` struct configured for primary actions with DPI-aware scaling.
60///
61/// ## Example
62/// ```rust
63/// let submit_button_style = create_primary_button_style();
64/// ```
65pub fn create_primary_button_style() -> ButtonStyle {
66    let scale = dpi_scale(1080.0); // Assuming a default window height for default values
67    ButtonStyle {
68        background_color: Color::rgb(30, 110, 30), // Slightly less saturated, dark mint green
69        hover_color: Color::rgb(25, 85, 25),       // Even darker, maintaining hue
70        pressed_color: Color::rgb(20, 65, 20),     // Darkest mint for pressed state
71        disabled_color: Color::rgb(110, 140, 110), // Muted, lighter mint for disabled state
72        border_color: Color::rgb(25, 85, 25),      // Matches hover color
73        border_width: 1.0,
74        corner_radius: 8.0,
75        padding: (16.0, 10.0),
76        text_style: TextStyle {
77            font_family: "HankenGrotesk".to_string(),
78            font_size: 18.0 * scale,
79            line_height: 20.0 * scale,
80            color: Color::rgb(255, 255, 255), // white
81            weight: Weight::MEDIUM,
82            style: Style::Normal,
83        },
84        text_align: TextAlign::Center,
85        spacing: ButtonSpacing::Hbar(0.3),
86    }
87}
